# 如何使用 Claude — 产研分享会网站大纲

> 面向受众：产品（0 AI 经验）、研发（1~2 个月 AI 经验）
> 学习路径：认知对齐 → 快速上手 → 进阶技巧 → 场景实战 → 团队协作 → 资源工具

---

## 文件组织约定

- 二级标题 → `content/` 下的数字前缀目录，如 `1.getting-started/`
- 三级标题 → 目录下的数字前缀 `.md` 文件，如 `1.index.md`
- 每个目录含 `.navigation.yml` 定义导航标题与排序
- `index.md` 为该章节的引言/概览页

---

## 01. 认知篇：Claude 能做什么

> **目标**：对齐认知，建立正确预期，避免"神话"或"妖魔化"AI
> **建议时长**：30 分钟

### 01. 为什么关注 Claude（`1.index.md`）
- AI 编程工具概览：ChatGPT、Claude、Copilot、Cursor 的差异化定位
- Claude 的核心特色：长上下文、代码能力、Artifacts、Claude Code
- 为什么产研团队现在就要关注：效率增益的真实数据
- 受众：全员

### 02. 能力边界：能做什么 vs 不能做什么（`2.capabilities.md`）
- Claude 的能力地图：代码生成、文档撰写、数据分析、逻辑推理
- 明确边界：无法替代业务判断、无法保证 100% 正确、无法访问实时信息（部分场景）
- 常见误区澄清："AI 会取代程序员"、"提示词万能"等
- 受众：全员

### 03. 产品与研发的学习曲线（`3.learning-curve.md`）
- 双视角对比：PM 从零开始 vs RD 已有 1-2 个月经验
- "蜜月期 → 瓶颈期 → 突破期"的典型路径
- 如何跨越"半懂不懂"的认知墙
- 受众：全员

---

## 02. 上手篇：第一次对话

> **目标**：零基础也能在 30 分钟内完成第一次有价值的 Claude 对话
> **建议时长**：45 分钟

### 01. 注册与界面导览（`1.index.md`）
- 账号注册流程（Claude.ai / Claude Code）
- 界面功能介绍：对话区、侧边栏、设置、Projects
- 基础操作：新建对话、复制回答、分享链接
- 受众：产品为主

### 02. 写好第一个提示词（`2.first-prompt.md`）
- 提示词的基本结构：任务 + 背景 + 约束
- "说人话"原则：如何像给同事布置任务一样描述需求
- 正反案例对比：模糊的提示词 vs 清晰的提示词
- 受众：全员

### 03. 对话管理与文件上传（`3.conversation-management.md`）
- 多轮对话技巧：如何保持上下文连贯
- 文件上传：代码文件、文档、图片的处理
- 对话历史管理：Projects 功能介绍
- 受众：全员

### 04. 产品专场：用 Claude 写 PRD（`4.pm-prd.md`）
- 从一句话想法到结构化 PRD 的完整流程
- 使用 Claude 辅助需求分析、用户故事撰写
- PRD 模板提示词示例
- 受众：产品

### 05. 研发专场：用 Claude 写第一行代码（`5.rd-first-code.md`）
- 代码生成：描述需求 → 获取代码 → 运行验证
- 代码解释：让 Claude 讲解陌生代码的工作原理
- 调试入门：描述报错 → 获取修复建议
- 受众：研发

## 03. 进阶篇：对话的艺术

> **目标**：掌握系统化提示词方法，让 Claude 输出质量稳定提升
> **建议时长**：45 分钟

### 01. 结构化提示词法则（`1.index.md`）
- 角色设定（Role）：让 Claude 扮演特定角色
- 背景信息（Context）：提供足够的业务/技术上下文
- 任务指令（Task）：明确具体要做什么
- 输出格式（Format）：规定返回的结构和样式
- 受众：全员

### 02. 让 Claude 更懂你的业务（`2.domain-knowledge.md`）
- 注入领域知识：项目背景、术语表、业务规则
- 使用 Projects 功能建立长期记忆
- 自定义指令（Custom Instructions）的配置与使用
- 受众：全员

### 03. 产品专场：PRD 质量检查清单（`3.pm-prd-quality.md`）
- 如何判断 AI 生成的 PRD 是否可用
- 常见 AI PRD 缺陷：逻辑自洽但脱离业务、遗漏边界条件
- 迭代优化 workflow：生成 → 审查 → 补充 → 再生成
- 受众：产品

### 04. 研发专场：模块化任务分解（`4.rd-task-decomposition.md`）
- 为什么大需求容易"翻车"：上下文超限与复杂度爆炸
- 模块化分解方法：按功能、按层级、按数据流拆分
- 子代理协作：让多个对话各司其职
- 受众：研发

### 05. 研发专场：Claude Code 工程化配置（`5.rd-claude-code-config.md`）
- `CLAUDE.md`：项目级配置文件的编写与最佳实践
- 自定义斜杠命令（Slash Commands）：固化常用操作
- Hooks 与 Skills：自动化重复性工作流
- 受众：研发

## 04. 实战篇：产研各环节的 Claude 应用

> **目标**：覆盖产研全流程的真实场景，提供可复用的操作模板
> **建议时长**：60 分钟

### 01. 需求分析与市场调研（`1.pm-research.md`）
- 竞品分析：让 Claude 辅助梳理竞品功能矩阵
- 用户反馈整理：从原始反馈中提取需求洞察
- 数据分析：用 Claude 解读数据报表、生成可视化建议
- 受众：产品

### 02. 原型验证与快速迭代（`2.pm-prototype.md`）
- 从 PRD 到可交互原型：Claude Artifacts 的使用
- 快速验证用户体验：生成演示代码、收集反馈、迭代优化
- "先别发给团队，用 AI 做个原型验证"方法论
- 受众：产品

### 03. 技术方案设计（`3.rd-tech-design.md`）
- 架构讨论：与 Claude 探讨技术选型、权衡分析
- 方案文档生成：从思路到可评审的技术文档
- 风险评估：让 Claude 帮助识别潜在技术风险
- 受众：研发

### 04. 编码实现：三种任务类型（`4.rd-coding.md`）
- BUG 修复：报错分析 → 根因定位 → 修复验证
- 新需求开发：从 0 到 1 的完整编码协作流程
- 功能重构：在现有代码基础上改进架构
- 每种任务的对话模式与注意事项
- 受众：研发

### 05. 代码审查与质量保障（`5.rd-code-review.md`）
- AI 辅助 Code Review：让 Claude 检查代码潜在问题
- 测试生成：自动生成单元测试、边界条件测试
- 质量 Checklist：AI 生成代码的安全审查要点
- 受众：研发

### 06. 文档与知识沉淀（`6.documentation.md`）
- 技术文档：API 文档、架构文档的自动生成与维护
- 会议纪要：从录音/速记到结构化纪要
- 知识库建设：将个人经验转化为团队可复用资产
- 受众：全员

## 05. 协作篇：AI 时代的产研协作

> **目标**：解决 AI 抹平技术鸿沟后的新协作摩擦，建立团队级规范
> **建议时长**：30 分钟

### 01. 当 PM 也会写代码：新协作标尺（`1.index.md`）
- 现象：PM 用 Claude 快速搭出演示版本，传统流程被打破
- 新分工边界：PM 负责"要什么"、RD 负责"怎么稳"、AI 负责"快验证"
- 如何重新定义交付物标准和责任归属
- 受众：全员

### 02. 原型驱动 vs 文档驱动（`2.prototype-vs-prd.md`）
- 需求评审新范式：原型演示 + 文档补充
- 如何平滑过渡：原型代码 → 生产代码的衔接策略
- 冲突解决：处理"PM 抢活干"的尴尬与误解
- 受众：全员

### 03. 团队级 AI 使用规范（`3.team-guidelines.md`）
- 编码标准：AI 生成代码必须符合团队现有规范
- 审查节点：区分"AI 辅助编写"和"AI 自动生成"的审查深度
- 安全策略：敏感数据处理、代码泄露风险、密钥管理
- 受众：全员

### 04. 知识沉淀：从个人经验到团队资产（`4.knowledge-sharing.md`）
- Skills 固化：将优秀提示词和经验沉淀为可复用模板
- 团队共享：Projects、CLAUDE.md 的团队化管理
- 持续迭代：建立反馈机制，不断优化 AI 协作流程
- 受众：全员

## 06. 资源篇：工具生态与持续学习

> **目标**：提供可立即上手的工具清单和持续学习的资源路径
> **建议时长**：15 分钟

### 01. Claude 生态工具一览（`1.index.md`）
- Artifacts：实时预览与交互
- Projects：长期上下文管理
- Claude Code：命令行编程助手
- MCP（Model Context Protocol）：连接外部工具生态
- 扩展插件与第三方集成
- 受众：全员

### 02. 推荐阅读与案例（`2.reading-list.md`）
- 官方文档与最佳实践指南
- 精选社区文章与真实案例分享
- 按角色分类的阅读路径：PM 路线 vs RD 路线
- 受众：全员

### 03. 常见问答与排障（`3.faq.md`）
- 高频问题速查：账号、限额、功能使用
- 常见报错处理：Claude Code 安装与配置问题
- 使用技巧合集：提升效率的快捷键与隐藏功能
- 受众：全员
